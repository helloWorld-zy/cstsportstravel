## User Scenarios & Testing

### User Story 1 - 出境游产品浏览与签证信息查询 (Priority: P1)

消费者可以浏览出境游跟团游产品，按国家/地区、签证类型等维度筛选，查看签证信息卡片（类型、办理周期、材料清单预览），并在出行前查询目的地入境政策和行前准备事项。

**Why this priority**: 出境游产品发现是出境游业务线的入口，签证信息是消费者决策的关键因素。与 MVP 境内游产品浏览（US2）共享搜索框架，叠加出境游特有筛选维度。

**Independent Test**: 用户可以从首页进入出境游频道，使用国家/地区和签证类型筛选产品，查看产品详情页中的签证信息卡片，浏览行前信息服务页面。

**Acceptance Scenarios**:

**后端 API 维度：**

1. **Given** 用户请求出境游产品列表, **When** 传入筛选条件（大洲→国家/地区层级树、签证类型：免签/落地签/电子签/需提前办签、出发城市含国际口岸、行程天数含15天以上）, **Then** 返回符合条件的产品列表，产品标签额外展示签证相关标签（免签直飞/落地签/含签证代办/拒签全退）
2. **Given** 用户请求出境游产品详情, **When** 传入产品 ID, **Then** 返回产品完整信息，在费用说明与退改政策之间包含签证信息卡片（签证类型、办理周期、建议最晚提交材料日期、按职业分类的材料清单预览、签证费用说明）
3. **Given** 用户请求签证信息, **When** 传入目的地国家, **Then** 返回该目的地的签证要求（免签/落地签/需提前办签）、入境政策、所需材料清单
4. **Given** 用户请求行前信息服务, **When** 传入目的地和出发日期, **Then** 返回入境材料清单、现金携带规定、违禁品清单、入境卡填写指引、海关申报指引、紧急联系信息
5. **Given** 用户已录入护照信息, **When** 搜索出境游产品, **Then** 系统前置校验护照有效期是否满足目的地要求，不满足的产品标注提示

**前端页面维度：**

6. **Given** 用户访问 Web 端出境游频道, **When** 浏览产品列表, **Then** 展示出境游专属筛选栏（大洲→国家/地区多选、签证类型标签筛选、国际出发口岸、行程天数含15天以上选项），产品卡片额外展示签证类型标签
7. **Given** 用户进入出境游产品详情页, **When** 浏览页面, **Then** 在费用说明与退改政策之间展示签证信息卡片（签证类型、办理周期、材料清单预览按在职/自由职业/退休/学生/儿童分类、签证费用），可点击展开完整材料清单；行程详情增加国际航班信息展示；FAQ 增加签证/外币/通讯等出境专属问题
8. **Given** 用户访问行前信息服务页, **When** 查看目的地信息, **Then** 展示入境政策查询、入境材料清单、现金携带规定、违禁品清单、入境卡中英文对照模板、海关申报指引、航班动态跟踪、目的地天气预报、时差显示、行前准备清单、紧急联系信息
9. **Given** 用户访问微信小程序出境游频道, **When** 浏览产品, **Then** 小程序版本适配移动端交互，出境游筛选和签证信息展示功能与 Web 端一致
10. **Given** 用户访问抖音小程序, **When** 浏览出境游产品, **Then** 抖音小程序通过条件编译适配抖音 API，核心页面（产品列表/详情/签证信息）功能与微信小程序一致

### User Story 2 - 出境游预订与签证代办 (Priority: P1)

消费者选择出境游产品后，在预订流程中填写含护照信息的出游人资料（系统校验护照有效期覆盖回程后6个月），可选择签证代办服务，支付成功后进入签证办理全流程跟踪。

**Why this priority**: 出境游预订闭环是出境游业务线的核心交易环节，签证服务交易闭环是与境内游的核心差异化能力。覆盖 PRD §4.3.4 表4-5 全部15个功能点。

**Independent Test**: 已登录且已实名的用户可以从出境游产品详情页完成"选团期→填出游人（含护照）→选签证代办→选附加服务→确认支付"全流程，支付成功后进入签证办理流程。

**Acceptance Scenarios**:

**后端 API 维度：**

1. **Given** 用户选择出境游团期和人数, **When** 提交预订请求, **Then** 系统实时计算总价（含签证代办费用），锁定库存，返回订单预览
2. **Given** 用户填写出游人信息, **When** 提交护照信息（姓名中文+拼音、护照号码、护照有效期、签发地、国籍）, **Then** 系统校验护照有效期覆盖回程日期后至少6个月，不通过则拦截并提示"您的护照有效期不足，建议换发后再预订"
3. **Given** 用户上传护照照片, **When** 系统处理图片, **Then** 通过 OCR 自动识别姓名、护照号、有效期、国籍并自动填充表单
4. **Given** 目的地需要提前办理签证, **When** 用户选择签证代办服务, **Then** 系统展示签证类型、费用和预计办理周期，签证费用作为子订单项计入总价
5. **Given** 目的地为免签或落地签国家, **When** 用户进入签证服务选择步骤, **Then** 系统展示免签/落地签说明，跳过签证代办选择
6. **Given** 支付成功后订单含签证代办服务, **When** 系统处理完成, **Then** 自动生成关联的签证服务订单，推送材料清单至用户
7. **Given** 用户提交签证材料, **When** 上传材料文件（护照扫描件、照片、在职证明、银行流水等，单文件≤10MB）, **Then** 系统存储文件并记录材料提交状态
8. **Given** 系统进行材料预审, **When** 检查材料完整性, **Then** 自动检查必填项是否齐全，人工审核合规性
9. **Given** 材料审核不通过, **When** 系统反馈结果, **Then** 逐条反馈修改意见，推送至用户（F-V-007）
10. **Given** 签证办理中, **When** 用户查询进度, **Then** 返回五节点状态机当前状态：待提交→审核中→已送签→已出签/已拒签，展示当前办理阶段和预计完成时间
11. **Given** 签证状态变更, **When** 系统处理状态流转, **Then** 通过短信+站内信双通道通知用户，更新订单详情页中的签证状态标签
12. **Given** 签证出签/拒签, **When** 系统处理结果, **Then** 第一时间通知用户，附签证页照片（出签）或拒签原因
13. **Given** 签证完成后护照需寄回, **When** 系统处理物流, **Then** 集成快递物流跟踪，用户可查看护照寄送状态
14. **Given** 用户查看历史签证记录, **When** 查询签证历史, **Then** 返回历史签证办理记录（签证类型、办理时间、出签结果）
15. **Given** 签证即将过期, **When** 距到期≤90天, **Then** 系统主动提醒用户签证有效期即将届满
16. **Given** 签证被拒签且产品标注"拒签全退", **When** 用户申请退款, **Then** 系统按拒签退款保障规则处理退款

**前端页面维度：**

17. **Given** 用户在出境游产品详情页点击"立即预订", **When** 进入预订流程, **Then** 进入五步向导页面（选团期→填出游人含护照→选签证代办→选附加服务→确认支付），护照信息表单包含姓名中文+拼音、护照号码、有效期、签发地、国籍字段，支持 OCR 识别自动填充
18. **Given** 用户在签证代办选择步骤, **When** 查看签证服务, **Then** 需提前办签产品展示签证类型、费用、办理周期和"需要签证代办"勾选项；免签/落地签产品展示相应说明并自动跳过此步骤
19. **Given** 用户在订单详情页查看签证进度, **When** 浏览签证状态, **Then** 展示五节点进度条（待提交→审核中→已送签→已出签/已拒签），当前节点高亮，已完成节点打勾，展示预计完成时间
20. **Given** 用户在订单详情页提交签证材料, **When** 上传材料, **Then** 展示按职业状态动态生成的材料清单（在职/自由职业/退休/学生/儿童），每项材料显示上传状态（待上传/已上传/审核通过/需补充），支持文件上传（≤10MB）
21. **Given** 材料审核不通过, **When** 用户查看反馈, **Then** 展示逐条修改意见，用户可补充材料后重新提交
22. **Given** 用户查看护照物流信息, **When** 浏览物流跟踪, **Then** 展示快递公司、运单号、物流轨迹时间线
23. **Given** 用户在微信小程序预订出境游, **When** 进入预订流程, **Then** 小程序版本的五步向导适配移动端交互，护照信息填写和签证代办选择功能与 Web 端一致
24. **Given** 用户在抖音小程序预订出境游, **When** 进入预订流程, **Then** 抖音小程序通过条件编译适配抖音支付 API，核心预订流程与微信小程序一致

### User Story 3 - 供应商入驻与产品管理 (Priority: P1)

供应商可以在线提交入驻申请（企业信息、资质文件），经平台审核通过后开通工作台账号，在工作台中发布和管理产品、处理订单、查看结算单和申请提现。运营人员可以审核供应商入驻申请、配置佣金规则。

**Why this priority**: 供应商开放平台是系统实现多边市场模式的核心支撑，是产品供给的基础。没有供应商入驻就没有产品可售。

**Independent Test**: 供应商可以提交入驻申请，运营人员可以审核通过/驳回，审核通过后供应商可以登录工作台发布产品、查看订单、查看结算单。

**Acceptance Scenarios**:

**后端 API 维度：**

1. **Given** 供应商提交入驻申请, **When** 填写企业信息（企业全称、统一社会信用代码、注册地址、注册资本、成立日期）和上传资质文件（营业执照、法人身份证、旅行社经营许可证等）, **Then** 系统校验资料完整性，生成入驻申请编号（APP-YYYYMMDD-NNNN），状态设为"待初审"
2. **Given** 运营专员初审, **When** 核查资料完整性与真实性, **Then** 初审在2个工作日内完成，通过则状态变为"待复审"，拒绝则填写原因并通知供应商
3. **Given** 运营主管复审, **When** 评估供应商经营能力与平台匹配度, **Then** 复审在初审通过后1个工作日内完成，通过则生成电子合同，拒绝则填写原因
4. **Given** 审核通过, **When** 系统生成合同, **Then** 按供应商类型生成合同模板，平台方和供应商方通过 CA 认证电子签章签署，合同生效后开通供应商工作台账号
5. **Given** 供应商登录工作台, **When** 发布产品, **Then** 填写基础信息→行程编辑→价格配置→退改规则→库存设置，产品编号格式为 OUT-{品类码}-{YYYYMMDD}-{序号}（出境游），提交后状态为"待审核"
6. **Given** 供应商编辑已上架产品关键字段, **When** 修改价格/行程/天数/退改规则, **Then** 产品状态变为"变更待审"，需重新审核
7. **Given** 供应商查看订单列表, **When** 筛选订单, **Then** 仅展示本供应商产品的订单（数据隔离），支持按状态/日期/产品筛选
8. **Given** 供应商处理退改申请, **When** 查看用户退改请求, **Then** 可同意或拒绝并填写处理意见
9. **Given** 结算周期到达, **When** 系统自动生成结算单, **Then** 汇总该周期内供应商的订单明细、退款明细和佣金计算结果，生成结算单
10. **Given** 供应商查看结算单, **When** 核对金额, **Then** 展示订单明细列表（订单号/产品名称/出行日期/订单金额/佣金比例/佣金金额/应收金额）、退款扣减明细、本期应结算总额
11. **Given** 供应商申请提现, **When** 提交提现请求, **Then** 选择提现金额与收款账户，提交平台审批

**前端页面维度：**

12. **Given** 供应商访问入驻申请页, **When** 填写申请表单, **Then** 展示多步骤表单（企业信息→资质上传→联系人信息→银行账户），支持表单自动保存，提交后展示申请编号和审核进度
13. **Given** 运营人员打开供应商入驻审核页, **When** 查看待审核列表, **Then** 展示待审核申请列表（支持按类型/时间/状态筛选），点击查看完整资料（含资质文件放大预览），操作按钮：通过/拒绝（需填写原因）/退回修改
14. **Given** 供应商登录工作台, **When** 打开产品管理页, **Then** 展示产品列表（支持按状态/名称/目的地筛选），点击"发布产品"进入多步骤表单（基础信息→行程编辑→价格配置→退改规则→库存设置），支持审核状态追踪
15. **Given** 供应商打开订单处理页, **When** 查看订单, **Then** 展示本供应商产品订单列表，支持确认/拒绝操作、查看详情、处理退改申请
16. **Given** 供应商打开财务结算页, **When** 查看结算单, **Then** 展示结算单列表（按结算周期），点击查看明细，支持确认结算单或提出异议，支持发起提现申请
17. **Given** 供应商打开收支明细页, **When** 查看财务数据, **Then** 展示每笔订单的平台佣金、应收金额、退款扣减等明细
18. **Given** 供应商打开数据统计页, **When** 查看经营数据, **Then** 展示销量统计、销售额趋势、评价数据

### User Story 4 - 二级分销体系 (Priority: P1)

分销商可以申请入驻（个人/企业两种类型），审核通过后在分销商中心为产品生成推广链接和二维码，查看佣金明细和申请提现。一级分销商可以邀请二级分销商加入团队。运营人员可以管理分销商、配置佣金规则、审核提现申请。

**Why this priority**: 分销体系是平台流量增长的核心引擎，二级分销模式在法律合规层面严格控制在两级以内。覆盖 PRD §8.7.1 佣金计算规则和 §8.7.2 防薅羊毛规则。

**Independent Test**: 分销商可以完成入驻申请→审核→协议签署→生成推广链接→查看佣金→提现全流程。一级分销商可以邀请二级分销商。运营人员可以管理分销商和审核提现。

**Acceptance Scenarios**:

**后端 API 维度：**

1. **Given** 个人分销商提交入驻申请, **When** 填写真实姓名、身份证号（上传正反面照片）、手机号、邮箱、结算银行卡信息、推广渠道说明, **Then** 系统自动校验身份证号校验码、银行卡号银联卡Bin校验，通过后进入待人工审核
2. **Given** 企业分销商提交入驻申请, **When** 填写企业全称、统一社会信用代码（上传营业执照）、法人信息、对公银行账户、联系人、资质证明, **Then** 系统自动校验营业执照工商信息，通过后进入待人工审核
3. **Given** 运营人员审核分销商申请, **When** 审核通过, **Then** 自动生成唯一分销编码（8位字母数字组合），发送审核通过通知，分销商状态为"待激活"
4. **Given** 分销商首次登录, **When** 完成协议签署（在线阅读并确认《分销商合作协议》）, **Then** 记录签署时间与IP地址，分销商状态变为"正常"
5. **Given** 一级分销商生成邀请链接/邀请码, **When** 分享给被邀请人, **Then** 被邀请人通过邀请链接/邀请码申请入驻，审核通过后自动建立二级分销关系（parent_id 关联）
6. **Given** 分销商为产品生成推广链接, **When** 选择目标产品, **Then** 生成专属短链接（含 distributor_code 参数），同时生成二维码图片（300×300/500×500/800×800px PNG，中心可嵌入平台Logo）
7. **Given** 消费者通过推广链接/二维码访问产品, **When** 首次点击, **Then** 系统将分销商编码写入Cookie/小程序本地存储（有效期30天），后续访问自动关联
8. **Given** 消费者通过推广链接下单并支付成功, **When** 系统处理订单, **Then** 订单记录 distributor_id_l1（直接推广者）和 distributor_id_l2（上级，可为NULL），异步计算佣金
9. **Given** 佣金计算完成, **When** 写入佣金明细, **Then** 一级佣金 = 实付金额 × 一级佣金比例（归属直接推广者），二级佣金 = 实付金额 × 二级佣金比例（归属上级一级分销商，若无上级则不产生），单笔订单总佣金不超过实付金额的50%
10. **Given** 佣金规则配置变更, **When** 运营人员修改佣金比例, **Then** 三级优先级体系：产品级 > 品类级 > 全局级，一级佣金比例 ≥ 二级佣金比例，配置5分钟内生效
11. **Given** 订单确认完成, **When** 佣金进入冻结期, **Then** 境内游 T+7天、出境游 T+15天、邮轮游 T+15天，冻结期内佣金不可提现
12. **Given** 冻结期届满, **When** 系统自动处理, **Then** 佣金状态从"已冻结"变为"可提现"
13. **Given** 订单发生退款, **When** 系统处理退佣, **Then** 全额退款全额追回佣金，部分退款按退款比例追回（应追回佣金 = 原佣金 × 退款金额 ÷ 实付金额），冻结期内退款直接调整，冻结期外退款从可提现余额扣除
14. **Given** 分销商申请提现, **When** 提交提现请求, **Then** 校验提现金额 ≥ 最低门槛（100元）、银行卡信息完整、分销商状态正常，进入审核队列
15. **Given** 运营人员审核提现, **When** 确认打款, **Then** 系统向财务系统发起打款指令，完成后佣金状态变为"已提现"，发送到账通知
16. **Given** 分销商存在自购行为, **When** 分销商点击自己的推广链接下单, **Then** 系统识别身份一致性，自动清除推广关联，该订单不计入分销业绩
17. **Given** 同一设备30天内关联多个账号, **When** 其中一个为分销商且其他账号通过其推广链接下单, **Then** 该订单不计入分销业绩（设备关联防薅羊毛规则）
18. **Given** 同一IP在1小时内通过同一推广链接产生超过10次点击, **When** 系统检测, **Then** 自动触发反作弊标记，后续该IP点击不计入推广数据

**前端页面维度：**

19. **Given** 用户访问分销商入驻申请页, **When** 填写申请表单, **Then** 展示个人/企业两种类型选择，个人类型表单包含姓名/身份证照片/手机号/邮箱/银行卡/推广渠道说明，企业类型表单包含企业信息/营业执照/法人信息/对公账户/资质证明，支持表单自动保存

### User Story 5 - 支付扩展（银联+定金尾款） (Priority: P2)

消费者可以使用银联支付（网关支付/WAP支付/云闪付），出境游和邮轮产品可选择定金+尾款支付模式（先付定金锁定库存，出行前N天付尾款），退款支持部分退款和原路退回。

**Why this priority**: 银联支付扩展支付渠道覆盖面，定金+尾款模式是高客单价产品的标准实践。两者均在 MVP 支付框架基础上扩展，不影响核心交易流程。

**Independent Test**: 消费者可以选择银联支付完成订单支付，出境游产品可以选择定金+尾款模式先付定金后付尾款，退款支持部分退款原路退回。

**Acceptance Scenarios**:

**后端 API 维度：**

1. **Given** 消费者选择银联支付, **When** 发起支付, **Then** 系统创建银联支付订单，PC端跳转银联网关页面（channelType=07），移动端跳转WAP支付页面（channelType=08），返回支付参数
2. **Given** 银联支付完成, **When** 系统收到后台通知（backUrl）, **Then** 验签通过后更新订单状态，以前台通知（frontUrl）仅作页面展示参考
3. **Given** 消费者选择定金+尾款模式, **When** 支付定金, **Then** 创建定金支付订单（金额=订单总额×定金比例，默认30%），支付成功后订单状态变为"已付定金"，库存锁定
4. **Given** 尾款开放期到达, **When** 系统处理提醒, **Then** 在尾款开放前3天通过短信、站内信、小程序消息三通道提醒用户
5. **Given** 消费者支付尾款, **When** 发起尾款支付, **Then** 创建尾款支付订单（金额=订单总额-已付定金），支付成功后订单状态变为"已付全款"
6. **Given** 尾款逾期未付, **When** 截止日后宽限期（24小时）结束, **Then** 系统自动取消订单，释放库存，按退改规则计算定金退还金额
7. **Given** 消费者申请部分退款, **When** 提交退款申请, **Then** 系统支持部分退款，退款金额不超过实付金额，原路退回用户原支付账户
8. **Given** 银联退款执行, **When** 调用退款接口, **Then** 当日交易通过消费撤销实时到账，隔日交易通过退货接口3-7个工作日到账

**前端页面维度：**

9. **Given** 消费者在支付页面选择支付方式, **When** 查看选项, **Then** 展示支付宝/微信支付/银联支付三种选择，银联支付支持网关支付和云闪付
10. **Given** 产品支持定金+尾款模式, **When** 消费者在确认订单页查看, **Then** 展示定金金额、尾款金额、尾款支付截止日期和逾期处理说明，可选择全额支付或定金+尾款模式
11. **Given** 消费者已付定金待付尾款, **When** 查看订单详情, **Then** 展示已付定金金额、应付尾款金额、尾款截止日期、倒计时提醒，提供"立即支付尾款"按钮
12. **Given** 消费者申请部分退款, **When** 填写退款申请, **Then** 展示退款金额预览（按退改规则自动计算）、退款原因选择、说明输入，提交后显示退款进度

### User Story 7 - 财务管理增强 (Priority: P2)

运营人员和财务人员可以在后台管理支付流水、退款、供应商结算单、发票，查看财务报表。财务管理覆盖支付渠道对账、供应商结算五步流程、退款分级审批、定金/尾款分账。

**Why this priority**: 财务管理是平台资金安全和合规运营的保障，依赖交易数据积累，可在核心交易流程稳定后完善。

**Independent Test**: 财务人员可以查看支付流水、审核退款申请、处理供应商结算单、管理发票、查看财务报表。

**Acceptance Scenarios**:

**后端 API 维度：**

1. **Given** 系统执行自动对账, **When** 每日凌晨2点触发, **Then** 从三渠道拉取前一日对账单，与本地流水逐笔核对，识别金额不一致、状态不一致、单边账等差异
2. **Given** 运营人员审核退款, **When** 查看退款申请, **Then** 系统展示按退改规则计算的应退金额、已发生费用、实际退款金额，退款分级审批：≤1000元运营审批、1000-5000元财务主管审批、>5000元总监审批
3. **Given** 结算周期到达, **When** 系统生成结算单, **Then** 执行五步流程：(1)自动生成结算单PDF（汇总订单明细/退款明细/佣金计算）→(2)供应商7日内核对确认或提出异议→(3)双方确认后进入付款审批（≤1万财务专员/1-5万财务主管/>5万总监联合审批）→(4)出纳执行银行转账并上传回单→(5)归档
4. **Given** 定金+尾款订单, **When** 财务人员查看流水, **Then** 分别记录定金收款、尾款收款和退款三类流水，标记款项类型，财务报表可独立统计
5. **Given** 用户申请发票, **When** 提交发票信息（抬头/税号/发票类型）, **Then** 运营人员审核通过后标记开票状态，支持增值税普通发票（电子）和增值税专用发票（电子），发票金额不超过订单实付金额

**前端页面维度：**

6. **Given** 财务人员打开支付流水页, **When** 查看流水, **Then** 展示所有渠道支付记录（支持按订单号/支付方式/支付状态/时间范围筛选），每笔流水展示支付金额/支付时间/渠道交易号/手续费
7. **Given** 财务人员打开退款管理页, **When** 审核退款, **Then** 展示退款申请列表（支持按金额区间/时间/状态筛选），退款详情展示应退金额/已发生费用/实际退款/退改规则匹配结果，操作按钮（通过/驳回需填写原因），金额超过审批阈值时提示需上级审批
8. **Given** 财务人员打开供应商结算页, **When** 处理结算单, **Then** 展示结算单列表（按供应商/结算周期筛选），查看详情（订单明细/退款扣减/佣金计算/应结算总额），操作：核对/审批/执行打款/上传回单
9. **Given** 财务人员打开发票管理页, **When** 处理发票申请, **Then** 展示发票申请列表，审核发票信息，标记开票状态，支持导出发票记录
10. **Given** 财务人员打开财务报表页, **When** 查看报表, **Then** 展示收入日报/月报/年报、产品收入分析、渠道收入分析、供应商付款报表、退款支出报表、毛利分析报表，支持按时间段导出Excel

**后端 API 维度：**

1. **Given** 用户通过抖音小程序登录, **When** 调用抖音登录API, **Then** 系统获取抖音 OpenID，首次登录自动创建平台账号并绑定
2. **Given** 用户在抖音小程序下单, **When** 发起支付, **Then** 系统创建支付订单，返回抖音小程序支付参数，前端调起抖音支付
3. **Given** 用户在抖音小程序查看订单, **When** 浏览订单列表, **Then** 返回该用户的订单数据，与微信小程序/Web端共享账号体系

**前端页面维度：**

4. **Given** 用户打开抖音小程序, **When** 进入登录页, **Then** 展示抖音授权登录入口，支持手机号绑定流程
5. **Given** 用户浏览产品列表, **When** 进入产品列表页, **Then** 展示产品列表（支持筛选/排序/搜索），适配抖音小程序UI规范
6. **Given** 用户查看产品详情, **When** 浏览详情页, **Then** 展示产品完整信息（图片轮播/行程/费用/退改/团期日历/评价），底部预订栏
7. **Given** 用户下单预订, **When** 进入预订流程, **Then** 展示预订向导（选团期→填出游人→附加服务→确认支付），适配抖音小程序交互
8. **Given** 用户查看订单, **When** 进入"我的订单", **Then** 展示订单列表（按状态Tab筛选），查看详情、申请退款
9. **Given** 用户查看个人中心, **When** 浏览个人页面, **Then** 展示个人信息、常用出游人管理、优惠券、分销商中心入口（如已注册分销商）

### Edge Cases

- **护照有效期边界**：用户护照有效期恰好覆盖回程后6个月（如回程7月1日，护照到期12月31日），系统应判定为通过（≥6个月）。
- **签证材料OCR识别失败**：护照照片模糊或角度不佳导致OCR识别失败，系统应允许用户手动填写并标注"未OCR识别"。
- **签证办理超时**：签证办理超过预计时间未出签，系统应自动推送延迟通知并建议用户联系客服。
- **供应商结算单异议**：供应商对结算单提出异议后，双方协商周期超过7天，系统应自动升级告警至运营经理。
- **分销商自购检测**：分销商使用不同设备但同一收货地址下单，系统应标记为疑似自购供人工复核。
- **佣金冻结期内退款**：订单在佣金冻结期内发生部分退款，系统应自动调整冻结中的佣金金额。
- **定金+尾款模式下全额退款**：用户已付定金+尾款后申请全额退款，系统应分别退还定金和尾款至原支付渠道。
- **银联支付回调延迟**：银联后台通知延迟超过30分钟，系统应主动调用银联查询接口确认交易状态。
- **优惠券与定金+尾款模式冲突**：使用优惠券的订单选择定金+尾款模式，优惠金额应在定金还是尾款中扣除？系统应在定金中扣除优惠金额。
- **分销商团队关系变更**：一级分销商被注销后，其下级二级分销商应转为平台直接管理的一级分销商。
- **抖音小程序与微信小程序账号打通**：同一手机号在微信小程序和抖音小程序登录，系统应自动关联到同一平台账号。
- **供应商入驻审核超时**：初审超过2个工作日未处理，系统应自动升级告警并通知运营经理。
- **多币种结算**：出境游涉及外币结算时，汇率以结算日中间价为准，系统应记录汇率快照。
- **并发库存扣减**：多人同时预订同一团期最后几个名额，系统采用库存预扣机制，先到先得，超时未支付自动释放。
- **退款到账户失败**：退款执行后用户账户未到账（如银行卡异常），系统应记录异常状态并通知运营人员人工处理。

- **FR-117**: 系统 MUST 支持签证历史记录查询（F-V-013）：用户可查看历史签证办理记录（签证类型/办理时间/出签结果）

**供应商开放平台模块**

- **FR-125**: 系统 MUST 支持供应商在线入驻申请：企业基础信息、营业执照、法人信息、经营范围、旅行社经营许可证、联系人信息、银行账户信息、资质补充文件
- **FR-126**: 系统 MUST 生成唯一入驻申请编号（APP-YYYYMMDD-NNNN），供应商可凭编号查询审核进度
- **FR-127**: 系统 MUST 支持供应商资质二级审核：运营专员初审（2个工作日内）→ 运营主管复审（初审通过后1个工作日内），超时自动升级告警
- **FR-128**: 系统 MUST 支持三种审核结果：通过（开通工作台账号）、拒绝（填写原因并通知）、退回修改（7日内可更新重新提交）
- **FR-129**: 系统 MUST 支持电子合同签署：CA认证电子签章，合同按供应商类型区分模板，签署完成后合同生效。签署超时处理：7日未签署自动提醒，30日未签署自动关闭入驻申请
- **FR-130**: 系统 MUST 提供供应商工作台，覆盖四大功能域：产品管理（发布/编辑/上下架/团期/库存/审核追踪）、订单处理（列表/确认/详情/退改）、财务结算（结算单查看/确认/提现/收支明细）、数据统计（销量/销售额/评价）
- **FR-131**: 系统 MUST 实施供应商数据隔离：通过 tenant_id + supplier_id 双重隔离，供应商仅能访问自身数据
- **FR-132**: 系统 MUST 支持供应商产品审核流程：供应商提交→运营审核（通过/驳回）→上架/退回修改，已上架产品关键字段变更触发重新审核
- **FR-133**: 系统 MUST 支持四级佣金比例配置（全局级<品类级<供应商级<产品级），支持阶梯佣金率（月销100单以下15%/100-500单13%/500单以上10%），供应商级佣金可覆盖品类默认值

- **FR-135**: 系统 MUST 支持结算周期配置：日结（T+1）/周结/月结，以供应商维度独立管理
- **FR-136**: 系统 MUST 支持供应商提现申请：选择提现金额与收款账户，提交平台审批

**二级分销模块（PRD §8 全部）**

- **FR-137**: 系统 MUST 支持分销商入驻申请：个人分销商（姓名/身份证/手机号/邮箱/银行卡/推广渠道）和企业分销商（企业信息/营业执照/法人/对公账户/联系人/资质），自动校验身份证号/营业执照/银行卡号
- **FR-138**: 系统 MUST 生成唯一分销编码（8位字母数字组合），用于推广链接参数标识
- **FR-139**: 系统 MUST 支持分销商审核：通过/拒绝/转补充材料（7日补充期限），审核通过后分销商状态为"待激活"
- **FR-140**: 系统 MUST 要求分销商首次登录完成协议签署（在线阅读并确认《分销商合作协议》），签署后状态变为"正常"。待激活状态超过15天未完成签署自动转为"已拒绝"
- **FR-141**: 系统 MUST 支持分销商等级体系：普通分销商（默认）和高级分销商（近90天≥50单且≥10万元且无违规，佣金上浮20%-50%，90天有效期到期复核）
- **FR-142**: 系统 MUST 支持四种分销商状态：正常/冻结/注销/待激活，冻结状态下推广链接失效、待结算佣金暂停结算
- **FR-143**: 系统 MUST 支持一级分销商通过邀请链接/邀请码发展二级分销商，被邀请人审核通过后自动建立二级分销关系
- **FR-144**: 系统 MUST 为每个分销商的每个产品生成专属推广链接（短链接）和二维码（300×300/500×500/800×800px PNG，中心可嵌入Logo）
- **FR-145**: 系统 MUST 采用URL参数优先+Cookie/本地存储备用的双轨跟踪策略，推广关联有效期30天
- **FR-146**: 系统 MUST 在订单中记录分销来源：distributor_id_l1（直接推广者）、distributor_id_l2（上级，可为NULL）、promotion_code、track_source
- **FR-147**: 系统 MUST 支持三级优先级佣金规则配置：产品级 > 品类级 > 全局级，一级佣金比例 ≥ 二级佣金比例，配置5分钟内生效
- **FR-148**: 系统 MUST 在订单支付成功时异步计算佣金：一级佣金 = 实付金额 × 一级比例（归属直接推广者），二级佣金 = 实付金额 × 二级比例（归属上级一级分销商），单笔订单总佣金不超过实付金额的50%
- **FR-149**: 系统 MUST 支持佣金T+N冻结期：境内游T+7天、出境游T+15天、邮轮游T+15天，冻结期内佣金不可提现
- **FR-150**: 系统 MUST 在订单退款时按比例追回佣金：全额退款全额追回，部分退款按退款比例追回，冻结期内退款直接调整，冻结期外退款从可提现余额扣除
- **FR-151**: 系统 MUST 支持分销商佣金提现：最低门槛100元，工作日24小时内审核，高级分销商享受加速结算（T+3）
- **FR-152**: 系统 MUST 实施防薅羊毛规则：自购禁止（分销商不能买自己推广的产品）、身份隔离（同一用户不能既是分销商又是其下级消费者）、设备关联（同一设备30天内关联多账号检测）、IP频率限制（同IP同链接1小时>10次点击触发反作弊）、违规处罚（警告/佣金扣除/冻结/注销）
- **FR-153**: 系统 MUST 在分销商中心提供数据概览看板：累计佣金、可提现金额、冻结中金额、快捷操作入口、最新公告
- **FR-154**: 系统 MUST 在"我的推广"模块展示推广产品列表、推广链接管理、推广数据统计（点击PV/UV/加购/转化率/成交订单/金额）
- **FR-155**: 系统 MUST 在"我的团队"模块（仅一级分销商）展示团队成员列表、团队业绩汇总、邀请功能、团队排行榜
- **FR-156**: 系统 MUST 在"佣金明细"模块展示佣金记录（来源订单/产品/品类/实付金额/佣金比例/金额/状态/结算时间），支持筛选和导出
- **FR-157**: 系统 MUST 在后台提供分销商管理：分销商列表（按类型/等级/状态筛选）、详情查看、等级调整、冻结/解冻、注销
- **FR-158**: 系统 MUST 在后台提供佣金结算审核：提现申请列表、确认打款（含批量）、拒绝提现
- **FR-159**: 系统 MUST 在后台提供分销数据报表：分销订单统计、佣金支出统计、分销商活跃度分析
- **FR-160**: 系统 MUST 在后台提供分销规则配置：三级佣金比例配置、结算规则配置（T+N/最低提现门槛/单日提现上限/自动结算开关）

**支付扩展模块**

- **FR-161**: 系统 MUST 支持银联支付：PC端网关支付（channelType=07）、移动端WAP支付（channelType=08）、云闪付APP支付，采用RSA-SHA256签名
- **FR-162**: 系统 MUST 以银联后台通知（backUrl）为确认交易成功的依据，前台通知（frontUrl）仅作页面展示参考
- **FR-163**: 系统 MUST 支持定金+尾款支付模式：运营人员配置定金比例（10%-50%，默认30%）和尾款支付截止时间
- **FR-164**: 系统 MUST 在定金支付成功后记录状态并标记订单为"已付定金"，在尾款开放前3天通过短信/站内信/小程序消息三通道提醒用户
- **FR-165**: 系统 MUST 在尾款逾期未付时（截止日后宽限期24小时结束）自动取消订单、释放库存、按退改规则计算定金退还金额
- **FR-166**: 系统 MUST 支持部分退款：退款金额不超过实付金额，原路退回用户原支付账户
- **FR-167**: 系统 MUST 支持银联退款：当日交易通过消费撤销实时到账，隔日交易通过退货接口3-7个工作日到账
- **FR-168**: 系统 MUST 在支付渠道回调丢失时主动查询支付状态（支付宝30秒内触发、后续60秒重试）

**营销系统模块**

- **FR-169**: 系统 MUST 支持四种优惠券类型：满减券（满X减Y）、折扣券（按比例折扣+上限）、现金券（无门槛减免）、兑换券（0元兑换指定商品）
- **FR-170**: 系统 MUST 支持优惠券配置参数：面额/折扣比例、最低消费门槛、总库存、每人限领、设备号限领、有效期（固定时段/领取后N天）、适用产品范围（全品类/指定品类/指定产品）、适用渠道、叠加规则
- **FR-171**: 系统 MUST 支持六种优惠券发放方式：系统推送、领券中心、商品挂载、活动赠送、分享裂变、兑换码
- **FR-172**: 系统 MUST 支持优惠券状态机：未开始→待使用→已占用→已使用/已过期/已退还/已作废
- **FR-173**: 系统 MUST 支持限时特惠活动：活动时间段、参与产品、秒杀价/特惠价、活动库存（与日常库存隔离）、限购数量
- **FR-174**: 系统 MUST 支持满减活动：阶梯满减（满200减20/满500减60）和统一满减，可设置活动范围和叠加规则
- **FR-175**: 系统 MUST 支持早鸟优惠：按提前预订时间设置阶梯折扣（提前60天8折/提前30天9折），系统自动匹配
- **FR-176**: 系统 MUST 支持优惠券效果分析：统计发放量、领取量、核销量、核销率和拉动GMV

**财务管理增强模块**

- **FR-177**: 系统 MUST 每日凌晨自动从三渠道拉取对账单，与本地流水逐笔核对，识别金额差异、状态差异、单边账
- **FR-178**: 系统 MUST 支持退款分级审批：≤1000元运营审批、1000-5000元财务主管审批、>5000元总监审批
- **FR-179**: 系统 MUST 支持供应商结算五步流程完整实现（生成→核对→确认→打款→归档）
- **FR-180**: 系统 MUST 分别记录定金收款、尾款收款和退款三类流水，标记款项类型
- **FR-181**: 系统 MUST 支持发票管理：增值税普通发票（电子）和增值税专用发票（电子），发票金额不超过订单实付金额
- **FR-182**: 系统 MUST 提供财务报表：收入日报/月报/年报、产品收入分析、渠道收入分析、供应商付款报表、退款支出报表、毛利分析报表

**抖音小程序模块**

- **FR-183**: 系统 MUST 支持抖音小程序登录：获取抖音OpenID，首次登录自动创建平台账号并绑定
- **FR-184**: 系统 MUST 支持抖音小程序支付：创建支付订单后返回抖音小程序支付参数
- **FR-185**: 抖音小程序 MUST 与微信小程序共享代码基，通过条件编译适配抖音API
- **FR-186**: 抖音小程序 MUST 支持核心页面：登录、产品列表/详情、预订、订单管理、个人中心

**架构变更**

- **FR-187**: 订单服务和支付服务 MUST 从单体拆分为独立部署单元
- **FR-188**: 系统 MUST 引入 NATS 消息队列处理异步事件（佣金计算、签证状态变更通知、对账任务）
- **FR-189**: 系统 MUST 引入 Meilisearch 替代数据库全文搜索

### Key Entities

- **签证订单 (VisaOrder)**: 关联主订单的签证服务订单，包含签证类型、办理状态（五节点状态机）、材料列表、审核意见、预计完成时间
- **签证材料 (VisaMaterial)**: 签证申请所需的各类文件，包含材料类型、文件路径、审核状态、审核意见，关联到签证订单
- **供应商 (Supplier)**: 入驻平台的第三方旅游服务商，包含企业信息、资质文件、入驻状态、佣金配置、结算周期
- **供应商结算单 (Settlement)**: 平台与供应商之间的资金清算记录，包含结算周期、订单明细、退款扣减、平台佣金、应付金额、结算状态
- **分销商 (Distributor)**: 推广平台产品的分销参与者，包含类型（个人/企业）、等级（普通/高级）、状态、分销编码、上级关系
- **分销关系 (DistributorRelation)**: 分销商之间的层级关系，包含分销商ID、上级ID、层级（1或2）、绑定时间、关系状态
- **佣金明细 (CommissionDetail)**: 分销订单的佣金记录，包含订单ID、分销商ID、佣金层级（一级/二级）、佣金金额、佣金状态（待结算/已冻结/可提现/已提现/已追回）
- **推广链接 (PromotionLink)**: 分销商的专属推广链接，包含分销商ID、产品ID、短链接、二维码路径、点击统计
- **优惠券 (Coupon)**: 平台发放的优惠凭证，包含类型、面额、使用条件、有效期、适用范围、库存、发放规则
- **优惠券领取记录 (CouponClaim)**: 用户领取优惠券的记录，包含用户ID、优惠券ID、领取时间、使用状态
- **促销活动 (Promotion)**: 平台的营销活动，包含类型（限时特惠/满减/早鸟优惠）、时间范围、参与产品、优惠规则、活动库存
- **支付流水 (PaymentTransaction)**: 所有渠道的支付记录，包含订单ID、支付渠道、支付金额、渠道交易号、支付状态、手续费
- **退款申请 (RefundRequest)**: 退款请求，关联订单，包含退款金额、退款原因、审核状态、退款状态机
- **发票 (Invoice)**: 用户的发票申请，包含发票类型、抬头、税号、金额、开票状态

## Success Criteria

### Measurable Outcomes

- **SC-001**: 出境游产品从浏览到签证材料提交全流程可在10分钟内完成
- **SC-002**: 签证材料OCR识别准确率≥90%（姓名、护照号、有效期三个字段全部正确）
- **SC-003**: 签证办理进度状态变更后，用户在5分钟内收到通知（短信+站内信）
- **SC-004**: 供应商入驻从提交申请到审核完成不超过3个工作日（初审2天+复审1天）
- **SC-005**: 供应商结算单从生成到双方确认不超过7个自然日
- **SC-006**: 分销佣金计算在订单支付成功后30秒内完成并可在佣金明细中查询
- **SC-007**: 分销商提现申请在工作日24小时内完成审核
- **SC-008**: 银联支付成功率≥97%（排除用户主动取消）
- **SC-009**: 定金+尾款模式下，尾款提醒在开放前3天准时送达率≥99%
- **SC-010**: 优惠券领取到使用的转化率≥30%
- **SC-011**: 防薅羊毛规则误判率（正常用户被误标记）<0.1%
- **SC-012**: 抖音小程序核心页面（登录/产品列表/详情/预订/订单）功能与微信小程序100%一致
- **SC-013**: 财务对账差异自动处理率≥80%（精度误差<0.01元和次日自动平账）
- **SC-014**: 一期上线后30天内，出境游订单不少于100笔，分销渠道贡献订单占比≥10%
- **SC-015**: 供应商工作台所有功能页面加载时间不超过2秒（P95）

## Assumptions

- **签证代办服务对接方式**：一期采用供应商人工处理签证服务，不对接第三方签证服务API。签证材料由供应商人工审核，签证进度由供应商手动更新状态。
- **护照OCR识别方案**：采用第三方OCR服务（如百度OCR/腾讯OCR），一期支持中国护照和常见国家护照识别，小语种护照降级为人工录入。
- **银联支付上线节奏**：银联支付在支付宝和微信支付稳定运行后上线，预计晚于一期核心功能2-4周。
- **抖音小程序审核周期**：抖音小程序需通过抖音平台审核，预计审核周期7-14天，需提前提交审核。
- **分销商佣金结算方式**：一期采用人工打款方式（企业网银转账），不对接银企直联接口。自动化打款作为二期优化。
- **供应商电子合同签署**：一期支持合同模板管理和PDF生成，电子签章对接第三方CA服务（如e签宝/法大大），具体服务商在实施计划阶段确定。
- **Meilisearch迁移范围**：一期将产品搜索从数据库全文搜索迁移至Meilisearch，订单和用户搜索暂不迁移。
- **NATS消息队列使用范围**：一期使用NATS处理佣金计算、签证状态变更通知、对账任务三类异步事件，后续逐步扩展。
- **定金+尾款模式适用产品**：一期仅出境游和邮轮产品支持定金+尾款模式，境内游仅支持全额支付。
- **分销商注销后佣金处理**：已入账佣金可正常提现，未结算佣金清除。注销后30天冷静期内可撤销。
- **优惠券与定金+尾款模式的优惠扣除**：使用优惠券的定金+尾款订单，优惠金额在定金中扣除，尾款按原价支付。
- **退款审批阈值与MVP一致**：≤1000元运营审批、1000-5000元财务主管审批、>5000元总监审批。
- **MVP功能延续**：一期完全兼容MVP已交付的所有功能，境内游产品浏览/预订/支付/订单管理/后台管理不受影响。
